# レビュー機能仕様

## 1. 概要

施設に対するレビューの閲覧・投稿機能を提供する。投稿にはログインが必要。レビュー0件の施設では「第一発見者」プロンプトを表示し、投稿のハードルを下げる。

## 2. データモデル

### Review

| フィールド | 型                | 必須 | 説明                                          |
| ---------- | ----------------- | ---- | --------------------------------------------- |
| id         | number (int)      | yes  | レビュー ID                                   |
| rating     | integer (1-5)     | yes  | 総合評価                                      |
| headline   | string            | no   | 一言コメント（最大 50 文字）                  |
| comment    | string            | no   | 詳細コメント（最大 1000 文字）                |
| userId     | string            | yes  | ユーザー ID（User.id と同値だが FK 制約なし） |
| museumId   | number (int)      | yes  | 施設 ID (FK → Museum)                         |
| userName   | string            | yes  | 投稿者名（User.name から取得して保存）        |
| createdAt  | string (ISO 8601) | yes  | 投稿日時                                      |

- 同一ユーザー × 同一施設の重複投稿は禁止（`@@unique([userId, museumId])`）

## 3. 設計方針

### headline（一言コメント）

- レビュー投稿のハードルを下げるための短文フィールド
- 50 文字制限により「書きやすさ」を実現（Twitter 的な制約の快感）
- レビューカードで最初に目に入る要素として表示
- OGP 画像やシェア時の引用テキストとしても活用予定

### userName の非正規化

- 投稿時に `User.name` を `userName` として保存する
- レビュー表示時に User テーブルへの JOIN が不要になり高速表示できる
- ユーザーが名前を変更しても過去レビューの投稿者名は変わらない（投稿時点の名前を保持）

### userId と User の関係

- Review.userId は User.id と同じ値を持つが、Prisma 上の FK リレーションは設定しない
- User 削除時にレビューを残す設計（非正規化の一環）

### 重複投稿の防止

- 同一ユーザーが同一施設に複数レビューを投稿することを DB 制約で防止する
- UI 側でもレビュー済みの場合はフォームを非表示にする

## 5. 機能

### レビュー閲覧

- 施設詳細ページ内にレビュー一覧を表示
- 一言コメント + 評価（星）+ 詳細コメント + ユーザー名 + 投稿日時

### レビュー投稿

- ログイン済み & 未レビュー: 投稿フォームを表示
- ログイン済み & レビュー済み: 「レビュー済みです」を表示
- 未ログイン: 「ログインしてレビューを書く」リンクを表示

### レビュー投稿フォーム

- 星評価セレクター（1〜5、必須）
- 一言コメント（任意、最大 50 文字、プレースホルダー例: 「時が止まった紡績工場」）
- 詳細コメントテキストエリア（任意、最大 1000 文字）
- 「投稿する」ボタン
- 投稿成功後、レビュー一覧を更新（SWR の mutate）

### レビュー 0 件時の「第一発見者」プロンプト

- レビューが 0 件の施設では、通常の「まだレビューがありません」の代わりに「第一発見者」プロンプトを表示
- 表示内容: 「最初のレビュアーになりませんか？」+ 「この施設にはまだレビューがありません。あなたの体験を共有しましょう。」
- ログイン済みの場合はそのまま投稿フォームを表示、未ログインの場合はログインリンクを表示

## 6. API

`docs/openapi.yaml` を参照。この機能に関連するエンドポイント:

- `POST /api/museums/{id}/reviews` — レビュー投稿（認証必須）
  - リクエストボディに `headline` フィールドを追加（任意、最大 50 文字）
