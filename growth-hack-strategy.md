# Museum Finder グロースハック戦略書

## 前提：コールドスタートの本質

Museum Finderが今直面している問題は「鶏と卵」の問題そのもの。レビューがないからユーザーが来ない、ユーザーが来ないからレビューが書かれない。この問題を解くには、**「鶏」と「卵」を同時に作る**か、**片方を自分で用意する**かの二択になる。

以下、3つの軸で「明日から実行可能」な施策を、優先度順に整理した。

---

## 1. 「0→1」の初期ユーザー獲得戦略

### 1-1. シードコンテンツを自分で仕込む（最優先・今週中）

レビューが0件のプラットフォームに人は書き込まない。食べログもRettyも初期はこれをやっている。

**具体的なアクション：**

- 自分自身で **最低30施設分のレビューを書く**。実際に訪問した施設のレビューを、写真付きで投稿する。1レビュー300〜500文字程度で十分
- 書けない施設は、経済産業省の「近代化産業遺産群33+33」のリストや、Wikipediaの記述を参考に **施設紹介文を充実させる**（レビューではなく施設説明として）
- レビューには「訪問年月」「おすすめ度」「所要時間」「アクセスの難易度」など、次の訪問者が本当に知りたい情報を含める

**なぜこれが最優先か：** 他のすべての施策でユーザーを呼び込んでも、中身がスカスカなら即離脱する。まず「来た人が価値を感じる状態」を作ることが大前提。

---

### 1-2. X（Twitter）での「漁場」特定とアプローチ（今週〜来週）

近代化産業遺産・企業博物館ファンは、X上にかなり濃いクラスタが存在する。

**狙うべきハッシュタグ：**

| ハッシュタグ              | 推定ボリューム | 特性                           |
| ------------------------- | -------------- | ------------------------------ |
| `#産業遺産`               | 中             | コア層。最もターゲットに近い   |
| `#廃墟` `#廃墟巡り`       | 大             | ボリューム大だが層がやや広い   |
| `#企業博物館`             | 小             | ニッチだが純度が高い           |
| `#近代化産業遺産`         | 小             | 制度に詳しい層。影響力が強い   |
| `#軍艦島` `#足尾銅山`     | 中             | スポット名は検索されやすい     |
| `#工場見学` `#博物館巡り` | 中             | 隣接層として有効               |
| `#廃線` `#鉱山`           | 中             | 鉄道・鉱業クラスタは熱量が高い |

**具体的なアクション：**

1. **Museum Finderの公式Xアカウントを作る**。プロフィールには「近代化産業遺産・企業博物館の探訪レビューサイト。エンジニアが個人開発中。」と明記する（個人開発であることはこの界隈では好感度が高い）
2. **自分の訪問記をスレッド形式で投稿する**。写真3〜4枚＋感想＋「詳しいレビューはこちら→（URL）」の導線。1日1スレッドを2週間続ける
3. **上記ハッシュタグで投稿しているアクティブユーザー50人をリストアップ**し、フォロー＋いいね＋引用RTで関係構築する。いきなりDMは送らない
4. **「近代化遺産 全国一斉公開 2025」（毎年10〜11月開催）** のタイミングに合わせた投稿企画を準備する。このイベントは2025年だけで167箇所が参加しており、参加者が訪問記を投稿する時期と重なる

---

### 1-3. ニッチコミュニティへのダイレクトアプローチ（来週〜）

**アプローチすべきコミュニティ：**

| コミュニティ                               | アプローチ方法                                                                                 |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| **ヘリタビ（heritabi.com）**               | 産業遺産を旅するサイト。相互リンク提案やコラボ記事の打診                                       |
| **産業考古学会**                           | 学術寄りだがメンバーは超コアファン。年会や学会誌で認知を取る                                   |
| **読書メーター「にわか歴史ファンの集い」** | 歴史ファンのコミュニティ。プロダクト紹介ではなく「こんな施設が面白かった」という形で自然に共有 |
| **note の産業遺産関連記事の著者**          | 訪問記を書いている人に個別でアプローチ。「あなたのレビューを掲載したい」は強い                 |
| **はてなブログの廃墟・博物館カテゴリ**     | 同上。ブロガーは自分の記事が別メディアで引用されることを好む                                   |

**重要な原則：** 「使ってください」ではなく **「あなたの知見を貸してください」** というアプローチにする。ニッチ領域のファンは「自分の知識が求められている」と感じると動く。

---

### 1-4. SEO戦略（中期・だが今から仕込む）

Museum Finderの最大のSEOアドバンテージは、**「施設名 + レビュー」「施設名 + 口コミ」のロングテールキーワードが誰にも取られていない**こと。食べログやGoogleマップがカバーしていない領域だからこそチャンスがある。

**具体的なアクション：**

- 各施設の詳細ページの `<title>` タグを `施設名 | レビュー・口コミ・見どころ | Museum Finder` の形式に統一
- 施設ごとに「概要」「見どころ」「アクセス」「所要時間」「レビュー」のセクションを設け、**構造化データ（Schema.org の LocalBusiness + Review）** をマークアップする
- 経済産業省認定の近代化産業遺産575+540件のリストを網羅的に登録し、**施設名での検索で確実にインデックスされる状態**を作る

## 2. 「最初のレビュー」を書かせるUI/UXの心理的仕掛け

### 2-1. 「空白恐怖」を逆手に取る

**「レビューがまだありません」は最悪の表示。** 代わりに、以下のように空白を「チャンス」に変える。

```
実装例：レビュー0件の施設ページ

┌──────────────────────────────────────┐
│  ★ この施設のレビュー                  │
│                                      │
│  🏆 最初のレビュアーになりませんか？     │
│                                      │
│  この施設にはまだレビューがありません。   │
│  あなたの体験を共有すると、             │
│  「第一発見者」バッジが付与されます。     │
│                                      │
│  [ レビューを書く → ]                  │
└──────────────────────────────────────┘
```

**ポイント：**

- 「第一発見者」「パイオニア」といった称号をつける。歴史ファンは「誰も行っていない場所を見つけた」ことに価値を感じる層
- レビュー0件の施設こそ目立たせる。「未踏の施設」一覧ページを作り、探検欲を刺激する

---

### 2-2. レビュー投稿のハードルを極限まで下げる

**自由記述だけのフォームは書けない。** 構造化された質問で誘導する。

```
実装例：レビュー投稿フォーム

┌──────────────────────────────────────┐
│  訪問日：2024年 __ 月（任意）           │
│                                      │
│  満足度：★★★★☆（タップで選択）        │
│                                      │
│  展示の充実度：★★★★★               │
│  アクセスしやすさ：★★★☆☆            │
│  穴場度：★★★★★                     │
│                                      │
│  一言で表すと？（20文字以内）           │
│  [ 時が止まった紡績工場               ]│
│                                      │
│  もう少し詳しく（任意）                │
│  [                                   ]│
│  [                                   ]│
│                                      │
│  写真を追加（任意）                    │
│  [ + 📷 ]                            │
│                                      │
│  [ 投稿する ]                         │
└──────────────────────────────────────┘
```

**設計思想：**

- **星評価だけでも投稿できる**ようにする。テキスト入力は任意。まず「数」を取る
- 「一言で表すと？」は20文字制限にすることで、逆に書きやすくなる（Twitter的な制約の快感）
- 「穴場度」という独自指標を入れる。これはGoogleマップにもTripAdvisorにもない、Museum Finder独自の価値になる
- 「展示の充実度」「アクセスしやすさ」などの軸は、次に来る人が本当に知りたい情報と一致させる

---

### 2-3. 投稿直後の「報酬」設計

レビュー投稿直後の画面が最も重要。ここで次の行動を誘発する。

```
実装例：投稿完了画面

┌──────────────────────────────────────┐
│  ✅ レビューを投稿しました！            │
│                                      │
│  🏆 「第一発見者」バッジを獲得！        │
│  📊 あなたの投稿数：3件               │
│  🗺️ 訪問エリア：関東・東北             │
│                                      │
│  ── 次に行きたい施設は？ ──           │
│                                      │
│  📍 近くの産業遺産                     │
│   ・旧○○紡績工場（レビュー2件）       │
│   ・△△鉱山資料館（まだレビューなし🏆） │
│                                      │
│  [ Xでシェアする ]  [ 次を探す ]       │
└──────────────────────────────────────┘
```

**ポイント：**

- 「レビューなし🏆」のマークで、「次もここのパイオニアになれる」と思わせる
- 近隣施設のレコメンドで、1人のユーザーから複数レビューを引き出す
- Xシェアボタンは、シェアテキストをあらかじめ生成しておく

### 2-4. プロフィールページをコレクション化する

歴史ファンは「踏破」「制覇」が好き。ユーザーのプロフィールページを、訪問記録のコレクションとして設計する。

```
実装例：プロフィールページ

┌──────────────────────────────────────┐
│  @history_walker の探訪記録            │
│                                      │
│  📊 訪問施設：12 / 1,115 施設         │
│  ✍️ レビュー：8件                     │
│  🏆 バッジ：第一発見者 ×3、鉱山マスター │
│                                      │
│  ── 訪問マップ ──                    │
│  [日本地図に訪問施設がプロットされる]    │
│                                      │
│  ── 訪問した施設カテゴリ ──           │
│  紡績 ████░░░░ 4施設                  │
│  鉱山 ██████░░ 6施設                  │
│  企業博物館 ██░░░░░░ 2施設             │
│                                      │
│  [ この記録をシェアする ]               │
└──────────────────────────────────────┘
```

**これがレビューを書く動機になる理由：** 訪問マップの「空白」を埋めたくなる。ゲーミフィケーションの「コレクション欲」と「完全制覇欲」を刺激する。

## 3. 開発力で殴るテクニカルグロースハック

### 3-1. OGP画像の動的生成（最優先・1〜2日で実装可能）

SNSでシェアされたときの見え方がすべてを決める。施設ごとに動的なOGP画像を生成する。

**実装方針：**

- Vercelの `@vercel/og` （Satori）を使い、Edge Functionで動的にOGP画像を生成する
- 施設名、カテゴリ、評価、レビュー件数を含むカード型のOGP画像を自動生成
- レビュー投稿時にも、ユーザー名・レビュー内容のサマリーを含むOGP画像を生成し、Xでのシェア時に「自分のレビューがカードとして表示される」体験を作る

```
実装例：OGP画像のレイアウト

┌──────────────────────────────────────┐
│  Museum Finder                       │
│                                      │
│  🏭 旧富岡製糸場                      │
│  カテゴリ：紡績・繊維                  │
│  ★★★★☆（4.2） レビュー 12件         │
│                                      │
│  「明治の息吹が残る、圧巻の空間」       │
│                                      │
│  museum-finder.vercel.app            │
└──────────────────────────────────────┘
```

**なぜこれが効くのか：** Xのタイムラインで「カード画像」が表示されるとCTRが劇的に上がる。特にレビューの一部が見えることで「読みたい」欲求を刺激する。

---

### 3-2. 構造化データの徹底実装（1日で実装可能）

GoogleがMuseum Finderの施設ページをリッチリザルトとして表示するようにする。

**実装すべきSchema.org：**

```json
{
  "@context": "https://schema.org",
  "@type": "TouristAttraction",
  "name": "旧富岡製糸場",
  "description": "...",
  "address": { "@type": "PostalAddress", "addressRegion": "群馬県", ... },
  "geo": { "@type": "GeoCoordinates", "latitude": "...", "longitude": "..." },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.2",
    "reviewCount": "12"
  },
  "review": [
    {
      "@type": "Review",
      "author": { "@type": "Person", "name": "..." },
      "reviewRating": { "@type": "Rating", "ratingValue": "5" },
      "reviewBody": "..."
    }
  ]
}
```

**効果：** Google検索結果に星評価が表示されるようになり、CTRが最大30%向上するという調査データがある。

### 3-3. 「訪問チェックイン」のURL自動生成（バイラルエンジン）

レビューを書かなくても、訪問したことをワンタップで記録＆シェアできる仕組み。

**仕組み：**

1. 施設ページに「ここに来た！」ボタンを設置
2. タップすると、位置情報（任意）＋日時でチェックインが記録される
3. チェックイン後、シェア用のURLが生成される

```
シェア用URL例：
museum-finder.vercel.app/checkin/abc123

→ このURLを開くと：
「@history_walker が 旧富岡製糸場 を訪問しました！」
+ 施設の基本情報カード
+ 「あなたも探訪する →」のCTA
```

**なぜこれが効くのか：** レビューを書くのは面倒だが、「来た」を記録するのは1秒。しかもそのシェアURLが新規ユーザーの入口になる。Swarmの仕組みに近い。

---

### 3-4. 施設データの自動充実化スクリプト（開発力の最大活用）

施設データが貧弱だとユーザーが定着しない。エンジニアとしてのスキルを活かし、以下を自動化する。

**やること：**

- 経済産業省の「近代化産業遺産群」認定リスト（575+540件）をスクレイピングしてデータベースに取り込む
- Wikipedia APIから各施設の基本情報（所在地、歴史、画像）を取得し、施設ページの初期データとして充填
- Google Maps APIで緯度経度・アクセス情報を取得し、地図表示に活用
- 各施設のSNS投稿数（Xでの言及数）をカウントし、「SNS注目度」として表示

**注意点：** 自動取得したデータは明確に「公開情報より引用」と明示し、ユーザーレビューとは区別する。

---

### 3-5. RSS / Atomフィードの生成（ニッチだが効果大）

歴史ファン・ブロガーはRSSリーダーを使っている率が一般より高い。

**実装すべきフィード：**

- `/feed/reviews` — 新着レビューのフィード
- `/feed/facilities/[category]` — カテゴリ別の新着施設フィード
- `/feed/user/[userId]` — 特定ユーザーの投稿フィード

**これにより：** はてなブログやnoteで「Museum Finderの新着レビュー」を自動取得して紹介してくれるブロガーが出てくる可能性がある。

---

### 3-6. 自動ツイートBotの実装（Xでの認知拡大）

Museum Finderの公式アカウントから、以下を自動投稿するBotを作る。

**投稿パターン：**

| トリガー               | 投稿内容                                                               |
| ---------------------- | ---------------------------------------------------------------------- |
| 新着レビュー           | 「📝 @user さんが ○○鉱山 をレビュー！ "一言コメント抜粋" → URL」       |
| 施設の「今日は何の日」 | 「📅 1901年3月13日、○○紡績工場が操業開始。現在は△△として公開中 → URL」 |
| 週間ランキング         | 「📊 今週のレビュー数TOP5！ 1位：○○ 2位：△△ ... → URL」                |

**「今日は何の日」が特に強い理由：** 歴史ファンはこの手のコンテンツが大好き。また、毎日投稿されるためフォロワーの目に入りやすく、施設の認知につながる。

---

各施設の訪問時に、コレクションカード風の画像を自動生成する。

実装例：施設カード

│                     │

**効果：** ユーザーが「自分だけのカードコレクション」をXやInstagramに投稿したくなる。毎回Museum FinderのブランドとURLが含まれるため、自然なバイラルが発生する。

## 最後に：このフェーズで最も重要なこと

第1形態で最も重要なのは **「100人の熱狂的なファン」を作ること** であり、1,000人の薄いユーザーではない。

そのためには：

1. **自分自身が最もアクティブなユーザーであること。** 毎週レビューを書き、Xで発信し、コミュニティに参加する
2. **最初の10人のレビュアーを「名前と顔がわかる関係」にすること。** DMで感謝を伝え、フィードバックを聞き、機能改善に反映する
